## Commands

```bash
npm run build     # prebuild (icon + rune images + templates) then webpack
npm run watch     # webpack watch mode (skips prebuild)
npm run serve     # serve dist/ at localhost:8080 for local testing
```

The `prebuild` step runs three scripts in sequence: `generate-icon.js`, `download-rune-images.js`, `generate-templates.js`. These only need to re-run if rune images or buff screenshots change; use `npm run watch` during normal development to skip them.

To test template accuracy:
```bash
node scripts/test-recognition.js
node scripts/test-recognition.js --verbose   # shows per-rune scores
```

## Architecture

A single-page Alt1 plugin that polls the player's buff bar to identify which rune the Runic Attuner is attuned to at each charge level, then builds a frequency table across 49 charge levels × 15 rune types.

All logic lives in **`src/index.ts`** (~606 lines). There is no framework or state management library — state is a single `data: FrequencyData` object persisted to `localStorage` under key `"runic-attuner-v2"`.

### Rune Identification Pipeline

1. **Buff detection** — `BuffReaderModule` (from `alt1/buffs`) finds the Runic Attuner buff icon in the player's buff bar.
2. **Template matching** — `buff.countMatch(template)` compares the live 25×25 inner icon against pre-built `ImageData` templates in `src/templates/*.data.png`. Templates are generated from 117 actual in-game buff screenshots (`src/attuner-buffs/`) using per-pixel variance: pixels that vary across captures (background bleed or charge-number text) are set transparent so `countMatch` skips them. Test accuracy against all 99 named samples: **100%**.
3. **Charge reading** — `buff.readArg("arg")` calls `OCR.readLine` with `pixel_8px_digits` at baseline `buffery+23`. The white charge number occupies icon rows 15-23. Returns the charge as a string; `parseInt` gives 1-49.
4. **Recording** — When the observed `(rune, charge)` pair changes between polls, a craft event is recorded. `data.lastCharge` is synced to the OCR reading; manual ± buttons allow correction if OCR fails.

### Prebuild Scripts (`scripts/`)

| Script | Purpose |
|---|---|
| `generate-icon.js` | Procedurally generates `src/icon.png` (pure Node, no deps) |
| `download-rune-images.js` | Fetches 15 rune PNGs from the RS wiki into `src/rune-images/` |
| `generate-templates.js` | Reads `src/attuner-buffs/*.png`, computes per-pixel variance per rune, writes 25×25 `src/templates/*.data.png` |
| `test-recognition.js` | Validates template matching against all 117 buff screenshots; run manually for QA |

`src/templates/` is regenerated by the prebuild and committed. `src/attuner-buffs/` contains the source game screenshots and is the ground truth for template quality.

### Build Output

Webpack bundles everything to `dist/main.js` (UMD, name `RunicAttuner`). The `appconfig.json` manifest declares `pixel` and `overlay` Alt1 permissions and a default window of 420×620px.
